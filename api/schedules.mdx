MELIKA's scheduling system is built around **franjas horarias** — fixed 40-minute availability blocks that doctors publish to open themselves up for patient bookings. Each franja represents one bookable slot on a specific date. Doctors manage their own franjas through the `/medico/franjas` routes and can inspect their booked schedule through two agenda views: a flat daily list and a FullCalendar-ready range response that encodes appointment state as colored events. All schedule endpoints require a valid doctor Bearer token.

---

## Franja Horaria Data Model

A `franja_horaria` record represents a single 40-minute time slot.

| Field | Type | Description |
|-------|------|-------------|
| `id` | `integer` | Unique identifier for the slot. |
| `id_medico` | `integer` | Foreign key — the owning doctor's ID. |
| `fecha` | `string` | Date of the slot in `YYYY-MM-DD` format. |
| `hora_inicio` | `string` | Slot start time in `HH:MM` format. |
| `hora_fin` | `string` | Slot end time in `HH:MM` format (exactly 40 minutes after `hora_inicio`). |
| `disponible` | `boolean` | `true` when no active appointment is attached; `false` once booked. |
| `created_at` | `string` | ISO 8601 timestamp of when the slot was created. |

<Warning>
  A slot can only be edited or deleted while `disponible` is `true`. Once a patient books an appointment against a franja, any attempt to modify or delete it will be rejected with a `400` error.
</Warning>

## Agenda Views

### Daily Agenda

```http
GET /medico/agenda?fecha=YYYY-MM-DD
```

Returns all appointments for the authenticated doctor on the specified date, ordered by `hora_inicio`. Ideal for displaying a single day's schedule in a list or timeline view. If `fecha` is omitted it defaults to today's date.

**Auth:** `verifyToken` + `isMedico`

#### Query Parameters

<ParamField query="fecha" type="string">
  The date to query in `YYYY-MM-DD` format (e.g. `2025-07-21`). Defaults to today if omitted.
</ParamField>

#### Response

An object containing the queried date and an array of appointment objects for that day.

<ResponseField name="fecha" type="string">
  The date that was queried (`YYYY-MM-DD`).
</ResponseField>

<ResponseField name="citas" type="array">
  Array of appointment objects for the requested day, ordered by `hora_inicio`.

  <Expandable title="Appointment object fields">
    <ResponseField name="id" type="integer">Appointment ID.</ResponseField>
    <ResponseField name="fecha" type="string">Date of the appointment (`YYYY-MM-DD`), formatted via SQL `TO_CHAR`.</ResponseField>
    <ResponseField name="hora_inicio" type="string">Appointment start time (`HH:MM`).</ResponseField>
    <ResponseField name="tipo_consulta" type="string">`presencial` or `teleconsulta`.</ResponseField>
    <ResponseField name="motivo" type="string">Patient's stated reason for the visit.</ResponseField>
    <ResponseField name="estado" type="string">Appointment state: `pendiente`, `completada`, `cancelada`, or `no_asistio`.</ResponseField>
    <ResponseField name="notas_medicas" type="string">Clinical notes added by the doctor, if any.</ResponseField>
    <ResponseField name="paciente_nombre" type="string">Patient's first name.</ResponseField>
    <ResponseField name="paciente_apellido" type="string">Patient's first surname.</ResponseField>
    <ResponseField name="paciente_telefono" type="string">Patient's phone number.</ResponseField>
    <ResponseField name="especialidad" type="string">Specialty name.</ResponseField>
    <ResponseField name="historia_id" type="integer">ID of the linked `historia_clinica` record, if one exists.</ResponseField>
  </Expandable>
</ResponseField>

#### Errors

| Status | Description |
|--------|-------------|
| `401`  | Missing or invalid Bearer token. |
| `403`  | Authenticated user is not a doctor. |
| `404`  | Doctor profile not found for the authenticated user. |
| `500`  | Internal server error. |

#### Example

<CodeGroup>
```bash cURL
curl -X GET "http://localhost:3000/medico/agenda?fecha=2025-07-21" \
  -H "Authorization: Bearer <medico_token>"
```

```json Response
{
  "fecha": "2025-07-21",
  "citas": [
    {
      "id": 42,
      "fecha": "2025-07-21",
      "hora_inicio": "09:00",
      "tipo_consulta": "presencial",
      "motivo": "Dolor de cabeza persistente",
      "estado": "pendiente",
      "notas_medicas": null,
      "paciente_nombre": "Sofía",
      "paciente_apellido": "Ramos",
      "paciente_telefono": "3009876543",
      "especialidad": "Neurología",
      "historia_id": null
    }
  ]
}
```
</CodeGroup>

### Range Agenda (FullCalendar)

```http
GET /medico/agenda/rango?inicio=YYYY-MM-DD&fin=YYYY-MM-DD
```

Returns all events within a date range as an array of **FullCalendar event objects**. The response merges two sets: booked appointments (excluding cancelled) and available unbooked franjas. Each event's color encodes its state, making it straightforward to render directly in a FullCalendar instance without client-side transformation.

**Auth:** `verifyToken` + `isMedico`

<Note>
  Both `inicio` and `fin` are required. Omitting either returns a `400` error. Available franjas appear first in the array, followed by booked appointment events.
</Note>

#### Color Reference

| State | Background | Border | Text | Meaning |
|-------|-----------|--------|------|---------|
| `disponible` | `#D1FAE5` | `#1A7A52` | `#065F46` | Slot is open and bookable. |
| `pendiente` | `#B45309` | `#92400E` | `#fff` | Appointment booked, not yet attended. |
| `completada` | `#1A7A52` | `#145C3E` | `#fff` | Appointment successfully completed. |
| `no_asistio` | `#6B7280` | `#4B5563` | `#fff` | Patient did not attend. |

#### Query Parameters

<ParamField query="inicio" type="string" required>
  Start of the date range in `YYYY-MM-DD` format (inclusive).
</ParamField>

<ParamField query="fin" type="string" required>
  End of the date range in `YYYY-MM-DD` format (inclusive).
</ParamField>

#### Response

An array of FullCalendar-compatible event objects. Available franja events appear before appointment events.

<ResponseField name="[]" type="array of objects">
  <Expandable title="Available franja event fields">
    <ResponseField name="id" type="string">Prefixed string ID in the format `"franja-{id}"` (e.g. `"franja-101"`).</ResponseField>
    <ResponseField name="title" type="string">`"🟢 Disponible (sin reservar)"`.</ResponseField>
    <ResponseField name="start" type="string">ISO datetime string: `"YYYY-MM-DDTHH:MM"` (no seconds).</ResponseField>
    <ResponseField name="end" type="string">ISO datetime string: `"YYYY-MM-DDTHH:MM"` (no seconds).</ResponseField>
    <ResponseField name="backgroundColor" type="string">`#D1FAE5` (light green).</ResponseField>
    <ResponseField name="borderColor" type="string">`#1A7A52` (dark green).</ResponseField>
    <ResponseField name="textColor" type="string">`#065F46` (very dark green).</ResponseField>
    <ResponseField name="display" type="string">Always `"block"`.</ResponseField>
    <ResponseField name="extendedProps" type="object">
      <Expandable title="extendedProps">
        <ResponseField name="esFranjaDisponible" type="boolean">Always `true` for available franjas.</ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>

  <Expandable title="Booked appointment event fields">
    <ResponseField name="id" type="string">Prefixed string ID in the format `"cita-{id}"` (e.g. `"cita-42"`).</ResponseField>
    <ResponseField name="title" type="string">Display label: `"{Estado} · {paciente_nombre} {paciente_apellido}"` (e.g. `"Pendiente · Sofía Ramos"`).</ResponseField>
    <ResponseField name="start" type="string">ISO datetime string: `"YYYY-MM-DDTHH:MM"` (no seconds).</ResponseField>
    <ResponseField name="backgroundColor" type="string">Hex fill color per appointment state (see color reference above).</ResponseField>
    <ResponseField name="borderColor" type="string">Darker hex border color per appointment state.</ResponseField>
    <ResponseField name="textColor" type="string">Always `"#fff"`.</ResponseField>
    <ResponseField name="extendedProps" type="object">
      <Expandable title="extendedProps">
        <ResponseField name="tipo" type="string">`presencial` or `teleconsulta`.</ResponseField>
        <ResponseField name="estado" type="string">Appointment state string.</ResponseField>
        <ResponseField name="esFranjaDisponible" type="boolean">Always `false` for booked appointments.</ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

#### Errors

| Status | Description |
|--------|-------------|
| `400`  | `inicio` or `fin` parameter is missing. |
| `401`  | Missing or invalid Bearer token. |
| `403`  | Authenticated user is not a doctor. |
| `404`  | Doctor profile not found for the authenticated user. |
| `500`  | Internal server error. |

#### Example

<CodeGroup>
```bash cURL
curl -X GET "http://localhost:3000/medico/agenda/rango?inicio=2025-07-21&fin=2025-07-25" \
  -H "Authorization: Bearer <medico_token>"
```

```json Response

    "id": "franja-102",
    "title": "🟢 Disponible (sin reservar)",
    "start": "2025-07-21T09:40",
    "end": "2025-07-21T10:20",
    "backgroundColor": "#D1FAE5",
    "borderColor": "#1A7A52",
    "textColor": "#065F46",
    "display": "block",
    "extendedProps": {
      "esFranjaDisponible": true
    }
  },
  {
    "id": "cita-42",
    "title": "Pendiente · Sofía Ramos",
    "start": "2025-07-21T09:00",
    "backgroundColor": "#B45309",
    "borderColor": "#92400E",
    "textColor": "#fff",
    "extendedProps": {
      "tipo": "presencial",
      "estado": "pendiente",
      "esFranjaDisponible": false
    }
  }
]
```
</CodeGroup>

## Slot Management

### List Slots by Date

```http
GET /medico/franjas?fecha=YYYY-MM-DD
```

Returns all `franja_horaria` records owned by the authenticated doctor. Optionally filtered to a specific date via the `fecha` query parameter. Each record includes a `cita_id` column — the ID of any non-cancelled appointment linked to that slot.

**Auth:** `verifyToken` + `isMedico`

#### Query Parameters

<ParamField query="fecha" type="string">
  Optional date filter in `YYYY-MM-DD` format. If omitted, all franjas for the doctor are returned.
</ParamField>

#### Response

Array of `franja_horaria` objects (see the [data model](#franja-horaria-data-model) above), each extended with:

<ResponseField name="cita_id" type="integer">
  ID of the active (non-cancelled) appointment linked to this slot. `null` when the slot is still available.
</ResponseField>

#### Errors

| Status | Description |
|--------|-------------|
| `401`  | Missing or invalid Bearer token. |
| `403`  | Authenticated user is not a doctor. |
| `404`  | Doctor profile not found for the authenticated user. |
| `500`  | Internal server error. |

#### Example

<CodeGroup>
```bash cURL
curl -X GET "http://localhost:3000/medico/franjas?fecha=2025-07-21" \
  -H "Authorization: Bearer <medico_token>"
```

```json Response
[
  {
    "id": 101,
    "id_medico": 7,
    "fecha": "2025-07-21",
    "hora_inicio": "09:00",
    "hora_fin": "09:40",
    "disponible": false,
    "cita_id": 42,
    "created_at": "2025-07-10T14:32:00.000Z"
  },
  {
    "id": 102,
    "id_medico": 7,
    "fecha": "2025-07-21",
    "hora_inicio": "09:40",
    "hora_fin": "10:20",
    "disponible": true,
    "cita_id": null,
    "created_at": "2025-07-10T14:32:05.000Z"
  }
]
```
</CodeGroup>

### Create Slots

```http
POST /medico/franjas
```

Generates multiple consecutive 40-minute availability slots from a given time window on a specific date. The controller internally calls `generarFranjasConDescanso`, splitting the `hora_inicio`→`hora_fin` range into 40-minute blocks and optionally skipping a break window. Slots that already exist for the same doctor, date, and start time are silently skipped (not duplicated).

**Auth:** `verifyToken` + `isMedico`

<Tip>
  This single endpoint replaces the need to create slots one by one. Supply the full working window (e.g. `08:00`→`17:00`) with an optional lunch break, and all 40-minute slots are generated in one request.
</Tip>

#### Request Body

<ParamField body="fecha" type="string" required>
  Date for the slots in `YYYY-MM-DD` format.
</ParamField>

<ParamField body="hora_inicio" type="string" required>
  Start of the working window in `HH:MM` format (e.g. `"08:00"`). Must be earlier than `hora_fin`.
</ParamField>

<ParamField body="hora_fin" type="string" required>
  End of the working window in `HH:MM` format (e.g. `"17:00"`).
</ParamField>

<ParamField body="inicio_descanso" type="string">
  Start of an optional break period in `HH:MM` format (e.g. `"12:00"`). Slots overlapping the break window are skipped.
</ParamField>

<ParamField body="fin_descanso" type="string">
  End of the break period in `HH:MM` format (e.g. `"13:00"`). Required when `inicio_descanso` is provided.
</ParamField>

#### Response `201`

<ResponseField name="mensaje" type="string">
  Human-readable summary (e.g. `"Se crearon 10 franja(s) de 40 minutos exitosamente (2 ya existían)."`).
</ResponseField>

<ResponseField name="insertadas" type="integer">
  Number of new slots actually inserted into the database.
</ResponseField>

<ResponseField name="duplicadas" type="integer">
  Number of slots skipped because they already existed for this doctor/date/hora_inicio combination.
</ResponseField>

<ResponseField name="franjas_intentadas" type="integer">
  Total number of 40-minute blocks the generator attempted to create from the supplied window.
</ResponseField>

#### Errors

| Status | Description |
|--------|-------------|
| `400`  | Missing required fields, `hora_inicio` ≥ `hora_fin`, or the time range produces no complete 40-minute blocks. |
| `401`  | Missing or invalid Bearer token. |
| `403`  | Authenticated user has no doctor profile. |
| `500`  | Internal server error. |

#### Example

<CodeGroup>
```bash cURL
curl -X POST http://localhost:3000/medico/franjas \
  -H "Authorization: Bearer <medico_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "fecha": "2025-07-22",
    "hora_inicio": "08:00",
    "hora_fin": "12:00",
    "inicio_descanso": "10:00",
    "fin_descanso": "10:20"
  }'
```

```json Response 201
{
  "mensaje": "Se crearon 5 franja(s) de 40 minutos exitosamente.",
  "insertadas": 5,
  "duplicadas": 0,
  "franjas_intentadas": 5
}
```
</CodeGroup>

### Update a Slot

```http
PATCH /medico/franjas/:id
```

Updates the date and/or time of an existing slot. Three constraints must all pass for the update to succeed:

1. The slot's `disponible` must be `true` (no active appointment attached).
2. The new `hora_fin` must be exactly 40 minutes after the new `hora_inicio`.
3. The new time range must not overlap with any other slot belonging to the same doctor on the same date.

**Auth:** `verifyToken` + `isMedico`

#### Path Parameters

<ParamField path="id" type="integer" required>
  The numeric ID of the `franja_horaria` to update.
</ParamField>

#### Request Body

<ParamField body="fecha" type="string" required>
  New date in `YYYY-MM-DD` format.
</ParamField>

<ParamField body="hora_inicio" type="string" required>
  New start time in `HH:MM` format.
</ParamField>

<ParamField body="hora_fin" type="string" required>
  New end time in `HH:MM` format. Must be exactly 40 minutes after `hora_inicio`.
</ParamField>

#### Response

<ResponseField name="mensaje" type="string">
  Confirmation: `"Franja actualizada correctamente."`
</ResponseField>

<ResponseField name="franja" type="object">
  The updated `franja_horaria` record (same shape as the list response).
</ResponseField>

#### Errors

| Status | Description |
|--------|-------------|
| `400`  | Missing required fields, `hora_inicio` ≥ `hora_fin`, duration is not exactly 40 minutes, or `disponible = false` (slot has an active appointment). |
| `401`  | Missing or invalid Bearer token. |
| `403`  | Authenticated user is not a doctor. |
| `404`  | No slot found with the given ID, or it belongs to another doctor. |
| `409`  | The new time range overlaps with another existing slot for this doctor and date. |
| `500`  | Internal server error. |

#### Example

<CodeGroup>
```bash cURL
curl -X PATCH http://localhost:3000/medico/franjas/115 \
  -H "Authorization: Bearer <medico_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "fecha": "2025-07-22",
    "hora_inicio": "11:00",
    "hora_fin": "11:40"
  }'
```

```json Response
{
  "mensaje": "Franja actualizada correctamente.",
  "franja": {
    "id": 115,
    "id_medico": 7,
    "fecha": "2025-07-22",
    "hora_inicio": "11:00",
    "hora_fin": "11:40",
    "disponible": true,
    "created_at": "2025-07-11T08:15:00.000Z"
  }
}
```
</CodeGroup>

### Delete a Slot

```http
DELETE /medico/franjas/:id
```

Permanently removes a time slot. The slot must have `disponible = true`; slots with an active (non-cancelled) appointment attached cannot be deleted.

**Auth:** `verifyToken` + `isMedico`

#### Path Parameters

<ParamField path="id" type="integer" required>
  The numeric ID of the `franja_horaria` to delete.
</ParamField>

#### Response

<ResponseField name="mensaje" type="string">
  `"Franja horaria eliminada correctamente."` on success.
</ResponseField>

#### Errors

| Status | Description |
|--------|-------------|
| `400`  | The slot has an active appointment (`disponible = false`): `"No puedes eliminar una franja con cita reservada."` |
| `401`  | Missing or invalid Bearer token. |
| `403`  | Authenticated user is not a doctor. |
| `404`  | No slot found with the given ID, or it belongs to another doctor. |
| `500`  | Internal server error. |

#### Example

<CodeGroup>
```bash cURL
curl -X DELETE http://localhost:3000/medico/franjas/115 \
  -H "Authorization: Bearer <medico_token>"
```

```json Response
{
  "mensaje": "Franja horaria eliminada correctamente."
}
```

```json Error — slot is booked
{
  "mensaje": "No puedes eliminar una franja con cita reservada."
}
```
</CodeGroup>
