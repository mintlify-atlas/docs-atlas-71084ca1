The Medications API exposes MELIKA's drug catalog for use across the platform — from patient-facing drug information pages to physician prescribing workflows. Medications are classified as either **OTC** (over-the-counter, no prescription required) or **Rx** (prescription-only), are linked to medical specialties for contextual filtering, and carry structured presentation data. All read endpoints are public with no authentication required, though write operations are restricted to admin routes at the application layer.

---

## OTC vs Rx Classification

<CardGroup cols={2}>
  <Card title="OTC — Over the Counter" icon="store">
    Medications available without a prescription. Suitable for display in patient-facing catalogs, symptom checkers, and general wellness flows. Filtered with `tipo=OTC`.
  </Card>
  <Card title="Rx — Prescription Only" icon="file-prescription">
    Medications that require a physician's prescription. These appear in clinical workflows, physician prescribing panels, and are referenced inside clinical records. Filtered with `tipo=Rx`.
  </Card>
</CardGroup>

## Data Model

The `medicamentos` table stores the full drug profile including commercial identity, active ingredient, lab origin, and specialty linkage.

| Field | Type | Notes |
|---|---|---|
| `id` | integer | Primary key |
| `nombre_comercial` | VARCHAR(150) | Brand/commercial name |
| `principio_activo` | VARCHAR(150) | Generic active ingredient name |
| `principio_activa` | VARCHAR(150) | Legacy duplicate of `principio_activo` — both are kept in sync on write |
| `laboratorio` | VARCHAR(100) | Manufacturing laboratory |
| `categoria` | VARCHAR(100) | Legacy text category field (superseded by `id_especialidad`) |
| `id_especialidad` | integer | FK → specialties (contextual linkage for catalog filtering) |
| `tipo` | ENUM | `OTC` or `Rx` (enforced by CHECK constraint) |
| `descripcion` | TEXT | Clinical description |
| `indicaciones` | TEXT | Clinical indications |
| `posologia` | TEXT | Dosage guidance |
| `contraindicaciones` | TEXT | Contraindications |
| `presentaciones` | TEXT | Available forms and dosages (e.g. tablets, syrup) |
| `registro_invima` | VARCHAR(50) | INVIMA registration number (unique when set) |
| `imagen_url` | VARCHAR(255) | URL to product image |
| `activo` | boolean | Only `TRUE` records appear in public queries. Default `TRUE` |
| `created_at` | timestamp | Auto-generated |
| `updated_at` | timestamp | Auto-updated |

<Note>
  All public-facing endpoints (`GET /medicamentos`, `GET /medicamentos/:id`) return only **active medications** (`activo = TRUE`). Inactive entries are accessible only via the admin route.
</Note>

## Endpoints

### GET /medicamentos

Returns a filtered list of active medications. Supports three independent query filters that can be combined freely: `tipo` restricts by prescription class, `id_especialidad` restricts by specialty linkage, and `buscar` performs a case-insensitive full-text search across both `nombre_comercial` and `principio_activo`. Results are ordered alphabetically by `nombre_comercial`.

**Authentication:** None (public)

<ParamField query="tipo" type="string">
  Filter by prescription classification. Accepted values: `OTC`, `Rx`. Omit to return both types.
</ParamField>
<ParamField query="id_especialidad" type="string">
  Filter by specialty. Pass a numeric specialty ID to restrict results to medications linked to that discipline. Pass `"Todos"` or omit entirely to return medications across all specialties.
</ParamField>
<ParamField query="buscar" type="string">
  Case-insensitive search string applied via `ILIKE` against both `nombre_comercial` and `principio_activo`. Partial matches are supported.
</ParamField>

<ResponseField name="[].id" type="integer">
  Medication identifier.
</ResponseField>
<ResponseField name="[].nombre_comercial" type="string">
  Commercial/brand name (e.g. `"Metformina 500"`).
</ResponseField>
<ResponseField name="[].principio_activo" type="string">
  Generic active ingredient (e.g. `"metformina"`).
</ResponseField>
<ResponseField name="[].laboratorio" type="string">
  Manufacturing laboratory name.
</ResponseField>
<ResponseField name="[].tipo" type="string">
  `"OTC"` or `"Rx"`.
</ResponseField>
<ResponseField name="[].descripcion" type="string">
  Clinical description of the medication.
</ResponseField>
<ResponseField name="[].presentaciones" type="string">
  Available forms and dosages.
</ResponseField>
<ResponseField name="[].imagen_url" type="string">
  URL to the product image.
</ResponseField>
<ResponseField name="[].id_especialidad" type="integer">
  ID of the linked specialty (nullable).
</ResponseField>
<ResponseField name="[].activo" type="boolean">
  Always `true` in public responses.
</ResponseField>

<CodeGroup>
```bash All active medications
curl -X GET "http://localhost:3000/medicamentos"
```

```bash Filter: Rx only
curl -X GET "http://localhost:3000/medicamentos?tipo=Rx"
```

```bash Filter: search by active ingredient
curl -X GET "http://localhost:3000/medicamentos?tipo=Rx&buscar=metformina"
```

```bash Filter: by specialty + type
curl -X GET "http://localhost:3000/medicamentos?tipo=OTC&id_especialidad=2"
```
</CodeGroup>

**Example — `GET /medicamentos?tipo=Rx&buscar=metformina`:**

```json
[
  {
    "id": 14,
    "nombre_comercial": "Metformina 500",
    "principio_activo": "metformina",
    "laboratorio": "Laboratorios GenFar",
    "tipo": "Rx",
    "descripcion": "Antidiabético oral de la clase biguanida. Reduce la producción hepática de glucosa.",
    "presentaciones": "Tabletas 500mg, 850mg, 1000mg",
    "imagen_url": "https://storage.example.com/meds/metformina.jpg",
    "id_especialidad": 4,
    "activo": true
  }
]
```

<Tip>
  Combine `tipo=Rx` with a `buscar` term to power a physician-side drug search widget during prescription creation.
</Tip>

### GET /medicamentos/admin

Returns all medications including inactive ones (`activo = FALSE`) for administrative management. Intended for use in admin dashboards where operators manage the drug catalog.

**Authentication:** None at route level (enforced at the admin layer)

<ResponseField name="[].id" type="integer">Medication identifier.</ResponseField>
<ResponseField name="[].nombre_comercial" type="string">Commercial name.</ResponseField>
<ResponseField name="[].principio_activo" type="string">Active ingredient.</ResponseField>
<ResponseField name="[].laboratorio" type="string">Manufacturing laboratory.</ResponseField>
<ResponseField name="[].tipo" type="string">`"OTC"` or `"Rx"`.</ResponseField>
<ResponseField name="[].activo" type="boolean">Active status — includes both `true` and `false` records.</ResponseField>

<CodeGroup>
```bash cURL
curl -X GET "http://localhost:3000/medicamentos/admin"
```
</CodeGroup>

### GET /medicamentos/categorias

Returns a deduplicated list of specialty **names** (not IDs) that have at least one active medication linked to them. This powers category filter UIs in medication browsers, letting patients or physicians narrow results to a specific discipline before applying further filters.

**Authentication:** None (public)

<ResponseField name="[]" type="string">
  Array of specialty name strings (e.g. `["Cardiología", "Endocrinología", "Pediatría"]`). Each value corresponds to the `nombre` field of a specialty that has at least one active medication linked via `id_especialidad`.
</ResponseField>

<CodeGroup>
```bash cURL
curl -X GET "http://localhost:3000/medicamentos/categorias"
```

```js JavaScript
const res = await fetch('http://localhost:3000/medicamentos/categorias');
const categories = await res.json();
// → ["Cardiología", "Endocrinología", "Neurología", "Pediatría"]
```
</CodeGroup>

**Example response:**

```json
[
  "Cardiología",
  "Endocrinología",
  "Medicina General",
  "Neurología",
  "Pediatría"
]
```

### GET /medicamentos/:id

Retrieves the full profile of a single active medication by its ID. Returns **404** if the medication does not exist or is inactive.

**Authentication:** None (public)

<ParamField path="id" type="integer" required>
  The medication's unique identifier.
</ParamField>

<ResponseField name="id" type="integer">Medication identifier.</ResponseField>
<ResponseField name="nombre_comercial" type="string">Commercial/brand name.</ResponseField>
<ResponseField name="principio_activo" type="string">Generic active ingredient.</ResponseField>
<ResponseField name="laboratorio" type="string">Manufacturing laboratory.</ResponseField>
<ResponseField name="tipo" type="string">`"OTC"` or `"Rx"`.</ResponseField>
<ResponseField name="descripcion" type="string">Clinical description.</ResponseField>
<ResponseField name="indicaciones" type="string">Clinical indications.</ResponseField>
<ResponseField name="contraindicaciones" type="string">Contraindications.</ResponseField>
<ResponseField name="presentaciones" type="string">Available forms and dosages.</ResponseField>
<ResponseField name="registro_invima" type="string">INVIMA registration number.</ResponseField>
<ResponseField name="imagen_url" type="string">Product image URL.</ResponseField>
<ResponseField name="id_especialidad" type="integer">Linked specialty ID.</ResponseField>
<ResponseField name="activo" type="boolean">Active status (`true` for all public responses).</ResponseField>
<ResponseField name="created_at" type="string">ISO 8601 creation timestamp.</ResponseField>
<ResponseField name="updated_at" type="string">ISO 8601 last-updated timestamp.</ResponseField>

<Expandable title="Error Responses">
  | Status | Condition |
  |---|---|
  | `400` | Provided `id` is not a valid number |

### POST /medicamentos

Creates a new medication entry in the catalog. The request body uses short field names (`nombre`, `principio`, `presentacion`) that the controller maps to the corresponding database column names. This route is exposed through the `/medicamentos` path and is also mirrored under `/admin/medicamentos` (with different field names — see the Admin API).

**Authentication:** None at route level (enforced at the admin layer)

<ParamField body="nombre" type="string" required>
  The brand or commercial name of the medication. Stored as `nombre_comercial`.
</ParamField>
<ParamField body="tipo" type="string" required>
  Prescription classification. Must be exactly `"OTC"` or `"Rx"`.
</ParamField>
<ParamField body="principio" type="string">
  Active ingredient(s). Stored in both `principio_activo` and `principio_activa` columns.
</ParamField>
<ParamField body="descripcion" type="string">
  Free-text description of the medication.
</ParamField>
<ParamField body="presentacion" type="string">
  Physical form / presentations (e.g. `"Tabletas 500 mg"`). Stored as `presentaciones`.
</ParamField>
<ParamField body="activo" type="boolean">
  Whether the medication is active in the catalog. Defaults to `true`.
</ParamField>
<ParamField body="laboratorio" type="string">
  Name of the pharmaceutical laboratory.
</ParamField>
<ParamField body="id_especialidad" type="integer">
  Optional link to a specialty ID (used for catalog filtering).
</ParamField>
<ParamField body="imagen_url" type="string">
  URL of the medication's product image.
</ParamField>

<CodeGroup>
```bash cURL
curl -X POST http://localhost:3000/medicamentos \
  -H "Content-Type: application/json" \
  -d '{
    "nombre": "Atorvastatina 20",
    "principio": "atorvastatina",
    "tipo": "Rx",
    "descripcion": "Inhibidor de la HMG-CoA reductasa. Indicado para la reducción del colesterol LDL.",
    "presentacion": "Tabletas 10mg, 20mg, 40mg",
    "laboratorio": "Pfizer",
    "id_especialidad": 1
  }'
```
</CodeGroup>

### PUT /medicamentos/:id

Fully updates an existing medication record. Uses the same short field names as `POST /medicamentos`. Setting `activo=false` removes the medication from all public-facing queries without deleting the record.

**Authentication:** None at route level (enforced at the admin layer)

<ParamField path="id" type="integer" required>
  The ID of the medication to update.
</ParamField>
<ParamField body="nombre" type="string">
  Updated commercial name. Stored as `nombre_comercial`.
</ParamField>
<ParamField body="principio" type="string">
  Updated active ingredient. Stored as `principio_activo` and `principio_activa`.
</ParamField>
<ParamField body="tipo" type="string">
  Updated classification: `"OTC"` or `"Rx"`.
</ParamField>
<ParamField body="descripcion" type="string">
  Updated clinical description.
</ParamField>
<ParamField body="presentacion" type="string">
  Updated presentation forms and dosages. Stored as `presentaciones`.
</ParamField>
<ParamField body="activo" type="boolean">
  Set to `false` to deactivate and hide from public listings.
</ParamField>
<ParamField body="laboratorio" type="string">
  Updated laboratory name.
</ParamField>
<ParamField body="id_especialidad" type="integer">
  Updated specialty linkage.
</ParamField>
<ParamField body="imagen_url" type="string">
  Updated product image URL.
</ParamField>

<CodeGroup>
```bash cURL
curl -X PUT http://localhost:3000/medicamentos/14 \
  -H "Content-Type: application/json" \
  -d '{
    "nombre": "Metformina 500",
    "principio": "metformina",
    "tipo": "Rx",
    "presentacion": "Tabletas 500mg, 850mg, 1000mg; Solución oral 500mg/5ml",
    "activo": true
  }'
```

```js JavaScript
const res = await fetch('http://localhost:3000/medicamentos/14', {
  method: 'PUT',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    nombre: 'Metformina 500',
    principio: 'metformina',
    tipo: 'Rx',
    presentacion: 'Tabletas 500mg, 850mg, 1000mg; Solución oral 500mg/5ml',
    activo: true
  })
});
const updated = await res.json();
```
</CodeGroup>
