The Admin API is the control plane of the MELIKA platform. It exposes a unified set of endpoints that let administrators monitor live statistics, manage user accounts, oversee appointments, build doctor schedules (individually or in bulk), and maintain the specialty and medication catalogs. Every endpoint in this group is protected by JWT verification and an admin-role guard — no operation can be performed by a patient or a doctor token.

<Warning>
  Admin tokens grant **full platform access** including the ability to deactivate users, cancel appointments, and modify the shared catalog. Store them in secrets managers, rotate them regularly, and never expose them in client-side code or public repositories.
</Warning>

<Note>
  All `/admin` routes require an `Authorization: Bearer <token>` header where the JWT payload contains `rol = 'admin'`. Requests without a valid token return `401 Unauthorized`; requests with a non-admin token return `403 Forbidden`.
</Note>

## Statistics

### Get Platform Statistics

Retrieve a real-time snapshot of the entire platform: aggregate counts of active records, appointment-state breakdown, the five most recent appointments, and the five most-requested doctors.

**`GET /admin/stats`**

No query parameters or request body required.

<CodeGroup>
```bash cURL
curl -X GET http://localhost:3000/admin/stats \
  -H "Authorization: Bearer <admin_token>"
```
</CodeGroup>

**Response `200 OK`**

<ResponseField name="totales" type="object">
  Aggregate counts across the platform.
  <Expandable title="totales fields">
    <ResponseField name="totales.pacientes" type="integer">
      Total **active** patient accounts (`rol = 'paciente'` AND `activo = TRUE`).
    </ResponseField>
    <ResponseField name="totales.medicos" type="integer">
      Total **active** doctor accounts (`activo = TRUE`).
    </ResponseField>
    <ResponseField name="totales.citas" type="integer">
      Total non-cancelled appointments (`estado NOT IN ('cancelada')`).
    </ResponseField>
    <ResponseField name="totales.citasHoy" type="integer">
      Number of non-cancelled appointments scheduled for today (server's local date).
    </ResponseField>
    <ResponseField name="totales.especialidades" type="integer">
      Total **active** specialty entries (`activa = TRUE`).
    </ResponseField>
    <ResponseField name="totales.medicamentos" type="integer">
      Total **active** medication entries (`activo = TRUE`).
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="citasPorEstado" type="array">
  Appointment counts grouped by status across all appointments.
  <Expandable title="citasPorEstado item fields">
    <ResponseField name="citasPorEstado[].estado" type="string">
      Appointment status value (e.g. `pendiente`, `completada`, `cancelada`, `no_asistio`).
    </ResponseField>
    <ResponseField name="citasPorEstado[].total" type="string">
      Count of appointments in this status (returned as a numeric string from `COUNT(*)`).
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="ultimasCitas" type="array">
  The five most recently created appointments across all doctors and patients.
  <Expandable title="ultimasCitas item fields">
    <ResponseField name="ultimasCitas[].id" type="integer">Appointment ID.</ResponseField>
    <ResponseField name="ultimasCitas[].fecha" type="string">Date of the appointment (`YYYY-MM-DD`).</ResponseField>
    <ResponseField name="ultimasCitas[].hora_inicio" type="string">Start time (`HH:MM:SS`).</ResponseField>
    <ResponseField name="ultimasCitas[].estado" type="string">Current appointment status.</ResponseField>
    <ResponseField name="ultimasCitas[].tipo_consulta" type="string">Consultation type (`presencial` or `teleconsulta`).</ResponseField>
    <ResponseField name="ultimasCitas[].paciente_nombre" type="string">Patient's first name.</ResponseField>
    <ResponseField name="ultimasCitas[].paciente_apellido" type="string">Patient's first surname.</ResponseField>
    <ResponseField name="ultimasCitas[].medico_nombre" type="string">Doctor's first name.</ResponseField>
    <ResponseField name="ultimasCitas[].medico_apellido" type="string">Doctor's first surname.</ResponseField>
    <ResponseField name="ultimasCitas[].especialidad" type="string">Specialty associated with the appointment.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="medicosMasSolicitados" type="array">
  The five doctors with the highest number of non-cancelled appointments.
  <Expandable title="medicosMasSolicitados item fields">
    <ResponseField name="medicosMasSolicitados[].nombre" type="string">Doctor's first name.</ResponseField>
    <ResponseField name="medicosMasSolicitados[].primer_apellido" type="string">Doctor's first surname.</ResponseField>
    <ResponseField name="medicosMasSolicitados[].especialidad" type="string">Doctor's specialty.</ResponseField>
    <ResponseField name="medicosMasSolicitados[].total_citas" type="string">Total non-cancelled appointments for this doctor (numeric string).</ResponseField>
  </Expandable>
</ResponseField>

## User Management

Administrators can list all users with flexible filtering and toggle any non-admin account between active and inactive states.

### List Users

**`GET /admin/usuarios`**

<ParamField query="rol" type="string">
  Filter by user role. Accepted values: `paciente`, `medico`, `admin`.
</ParamField>

<ParamField query="buscar" type="string">
  Case-insensitive substring search (`ILIKE`) applied to `nombre`, `primer_apellido`, and `email`.
</ParamField>

<ParamField query="activo" type="string">
  Filter by account status. Accepted values: `true`, `false`.
</ParamField>

Results are ordered by `created_at` descending (newest accounts first).

<CodeGroup>
```bash cURL — list all doctors
curl -X GET "http://localhost:3000/admin/usuarios?rol=medico&activo=true" \
  -H "Authorization: Bearer <admin_token>"
```

```bash cURL — search by name
curl -X GET "http://localhost:3000/admin/usuarios?buscar=garcia" \
  -H "Authorization: Bearer <admin_token>"
```
</CodeGroup>

**Response `200 OK`** — array of user objects.

<ResponseField name="[]" type="array">
  <Expandable title="User object fields">
    <ResponseField name="id" type="integer">User ID.</ResponseField>
    <ResponseField name="nombre" type="string">First name.</ResponseField>
    <ResponseField name="primer_apellido" type="string">First surname.</ResponseField>
    <ResponseField name="email" type="string">Email address.</ResponseField>
    <ResponseField name="rol" type="string">User role: `paciente`, `medico`, or `admin`.</ResponseField>
    <ResponseField name="activo" type="boolean">Whether the account is active.</ResponseField>
    <ResponseField name="verificado" type="boolean">Whether the email has been verified.</ResponseField>
    <ResponseField name="telefono" type="string">Phone number (may be `null`).</ResponseField>
    <ResponseField name="ciudad" type="string">City (may be `null`).</ResponseField>
    <ResponseField name="created_at" type="string">ISO 8601 timestamp of account creation.</ResponseField>
  </Expandable>
</ResponseField>

### Toggle User Status

**`PATCH /admin/usuarios/:id/estado`**

Toggles the `activo` field of the specified user. If the user is currently active they will be deactivated, and vice versa.

<ParamField path="id" type="integer" required>
  ID of the user to toggle.
</ParamField>

<Warning>
  Admin accounts **cannot** be deactivated through this endpoint. Attempting to do so returns `400 Bad Request`.
</Warning>

<CodeGroup>
```bash cURL
curl -X PATCH http://localhost:3000/admin/usuarios/42/estado \
  -H "Authorization: Bearer <admin_token>"
```
</CodeGroup>

**Response `200 OK`**

<ResponseField name="mensaje" type="string">
  Human-readable confirmation: `"Usuario activado."` or `"Usuario desactivado."`.
</ResponseField>

**Error responses**

| Status | Condition |
|--------|-----------|
| `400`  | Attempted to deactivate an admin account. |
| `404`  | No user found with the given ID. |

## Appointment Management

Administrators have a global view of every appointment and can update any appointment's status, including releasing the associated time slot when a cancellation occurs.

### List Appointments

**`GET /admin/citas`**

<ParamField query="estado" type="string">
  Filter by appointment status. Accepted values: `pendiente`, `completada`, `cancelada`, `no_asistio`.
</ParamField>

<ParamField query="fecha_desde" type="string">
  Lower date bound (inclusive), format `YYYY-MM-DD`.
</ParamField>

<ParamField query="fecha_hasta" type="string">
  Upper date bound (inclusive), format `YYYY-MM-DD`.
</ParamField>

<ParamField query="buscar" type="string">
  Case-insensitive substring search (`ILIKE`) on patient and doctor names.
</ParamField>

Results are ordered by `fecha DESC`, `hora_inicio DESC`.

<CodeGroup>
```bash cURL — appointments for a date range
curl -X GET "http://localhost:3000/admin/citas?fecha_desde=2025-01-01&fecha_hasta=2025-01-31&estado=pendiente" \
  -H "Authorization: Bearer <admin_token>"
```
</CodeGroup>

**Response `200 OK`** — array of full appointment objects including patient name, doctor name, specialty, and time-slot end time (`hora_fin`).

---

### Update Appointment Status

**`PATCH /admin/citas/:id/estado`**

<ParamField path="id" type="integer" required>
  ID of the appointment to update.
</ParamField>

<ParamField body="estado" type="string" required>
  New status. Accepted values: `pendiente`, `completada`, `cancelada`, `no_asistio`.
</ParamField>

<ParamField body="razon_cancelacion" type="string">
  Optional cancellation reason. Recommended when `estado` is `cancelada`.
</ParamField>

<Tip>
  When `estado` is set to `cancelada`, the linked time slot (`franja_horaria`) is automatically released by setting `disponible = TRUE`, making it bookable again by other patients.
</Tip>

<CodeGroup>
```bash cURL — cancel an appointment
curl -X PATCH http://localhost:3000/admin/citas/88/estado \
  -H "Authorization: Bearer <admin_token>" \
  -H "Content-Type: application/json" \
  -d '{"estado": "cancelada", "razon_cancelacion": "Doctor unavailable"}'
```

```bash cURL — mark as completed
curl -X PATCH http://localhost:3000/admin/citas/88/estado \
  -H "Authorization: Bearer <admin_token>" \
  -H "Content-Type: application/json" \
  -d '{"estado": "completada"}'
```
</CodeGroup>

**Response `200 OK`**

<ResponseField name="mensaje" type="string">
  Human-readable confirmation of the status change.
</ResponseField>

**Error responses**

| Status | Condition |
|--------|-----------|
| `400`  | Invalid `estado` value supplied. |
| `404`  | No appointment found with the given ID. |

## Schedule Management

Administrators can create, update, and delete individual time slots (`franjas_horarias`) for any doctor, or generate an entire day's schedule in a single bulk request. All slots are exactly **40 minutes** long (`DURACION_FRANJA_MIN = 40`).

### Get Schedules (FullCalendar format)

**`GET /admin/horarios`**

Returns schedule events formatted for [FullCalendar](https://fullcalendar.io/). Each event represents one `franja_horaria` and is color-coded by availability/appointment status.

<ParamField query="inicio" type="string">
  Start date of the range to fetch, format `YYYY-MM-DD`.
</ParamField>

<ParamField query="fin" type="string">
  End date of the range to fetch, format `YYYY-MM-DD`.
</ParamField>

<ParamField query="id_medico" type="integer">
  Filter events to a specific doctor.
</ParamField>

<CodeGroup>
```bash cURL
curl -X GET "http://localhost:3000/admin/horarios?inicio=2025-06-01&fin=2025-06-07&id_medico=5" \
  -H "Authorization: Bearer <admin_token>"
```
</CodeGroup>

**Response `200 OK`** — array of FullCalendar event objects.

<ResponseField name="[]" type="array">
  <Expandable title="Event object fields">
    <ResponseField name="id" type="integer">Franja ID.</ResponseField>
    <ResponseField name="title" type="string">Display title for the calendar event. Available slots show `"🟢 Dr(a). [name] — Disponible"`; booked slots show the appointment status label and patient name.</ResponseField>
    <ResponseField name="start" type="string">ISO 8601 start datetime (e.g. `"2025-06-01T09:00:00"`).</ResponseField>
    <ResponseField name="end" type="string">ISO 8601 end datetime (e.g. `"2025-06-01T09:40:00"`).</ResponseField>
    <ResponseField name="backgroundColor" type="string">Hex color for the event background. Green (`#1A7A52`) when available; amber (`#B45309`) for pending, green (`#1A7A52`) for completed, grey (`#6B7280`) for no-show, or orange for other booked states.</ResponseField>
    <ResponseField name="borderColor" type="string">Hex color for the event border.</ResponseField>
    <ResponseField name="extendedProps" type="object">
      Additional metadata for the event.
      <Expandable title="extendedProps fields">
        <ResponseField name="extendedProps.disponible" type="boolean">Whether the slot is currently bookable.</ResponseField>
        <ResponseField name="extendedProps.id_medico" type="integer">ID of the doctor this slot belongs to.</ResponseField>
        <ResponseField name="extendedProps.medico_nombre" type="string">Full name of the doctor (`nombre primer_apellido`).</ResponseField>
        <ResponseField name="extendedProps.especialidad" type="string">Doctor's specialty.</ResponseField>
        <ResponseField name="extendedProps.cita_id" type="integer">ID of the linked appointment, if booked.</ResponseField>
        <ResponseField name="extendedProps.cita_estado" type="string">Status key of the linked appointment (`pendiente`, `completada`, `no_asistio`).</ResponseField>
        <ResponseField name="extendedProps.cita_estado_label" type="string">Human-readable label: `"Pendiente"`, `"Completada"`, or `"No asistió"`.</ResponseField>
        <ResponseField name="extendedProps.paciente" type="string">Patient's full name, if the slot is booked. `null` when available.</ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

### Create a Single Slot

**`POST /admin/horarios`**

Creates one time slot for a doctor. The slot duration is not validated server-side on this route — use `POST /admin/horarios/masivo` to guarantee 40-minute alignment.

<ParamField body="id_medico" type="integer" required>
  ID of the doctor this slot belongs to.
</ParamField>

<ParamField body="fecha" type="string" required>
  Date of the slot, format `YYYY-MM-DD`.
</ParamField>

<ParamField body="hora_inicio" type="string" required>
  Start time of the slot, format `HH:MM`.
</ParamField>

<ParamField body="hora_fin" type="string" required>
  End time of the slot, format `HH:MM`. Should be exactly 40 minutes after `hora_inicio`.
</ParamField>

<CodeGroup>
```bash cURL
curl -X POST http://localhost:3000/admin/horarios \
  -H "Authorization: Bearer <admin_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "id_medico": 5,
    "fecha": "2025-06-10",
    "hora_inicio": "09:00",
    "hora_fin": "09:40"
  }'
```
</CodeGroup>

**Response `201 Created`**

<ResponseField name="mensaje" type="string">
  `"Franja creada."`
</ResponseField>

<ResponseField name="franja" type="object">
  The newly created slot.
  <Expandable title="franja fields">
    <ResponseField name="franja.id" type="integer">Slot ID.</ResponseField>
    <ResponseField name="franja.id_medico" type="integer">Doctor ID.</ResponseField>
    <ResponseField name="franja.fecha" type="string">Date.</ResponseField>
    <ResponseField name="franja.hora_inicio" type="string">Start time.</ResponseField>
    <ResponseField name="franja.hora_fin" type="string">End time.</ResponseField>
    <ResponseField name="franja.disponible" type="boolean">Always `true` for a newly created slot.</ResponseField>
  </Expandable>
</ResponseField>

**Error responses**

| Status | Condition |
|--------|-----------|
| `404`  | Doctor not found or inactive. |
| `409`  | A slot with the same doctor, date, and time already exists (unique constraint). |

### Bulk Generate Slots

**`POST /admin/horarios/masivo`**

Automatically generates all 40-minute slots that fit within a workday window, optionally skipping a break period. This is the recommended way to populate a full day's schedule.

<Info>
  Slot duration is fixed at **40 minutes** (`DURACION_FRANJA_MIN = 40`). The generator advances the cursor by 40 minutes at a time from `hora_inicio` until there is no room for another full slot before `hora_fin`. Any window that overlaps with `inicio_descanso`–`fin_descanso` is skipped entirely.
</Info>

<ParamField body="id_medico" type="integer" required>
  ID of the doctor for whom slots are generated.
</ParamField>

<ParamField body="fecha" type="string" required>
  Date for the generated slots, format `YYYY-MM-DD`.
</ParamField>

<ParamField body="hora_inicio" type="string" required>
  Workday start time, format `HH:MM`.
</ParamField>

<ParamField body="hora_fin" type="string" required>
  Workday end time, format `HH:MM`. The range must accommodate at least one 40-minute slot.
</ParamField>

<ParamField body="inicio_descanso" type="string">
  Break start time, format `HH:MM`. Optional. Must fall strictly within the workday window.
</ParamField>

<ParamField body="fin_descanso" type="string">
  Break end time, format `HH:MM`. Optional. Must fall strictly within the workday window and be after `inicio_descanso`.
</ParamField>

**Full workday example — 08:00 to 17:00 with a lunch break from 12:00 to 13:00**

The generator produces the following slots (each 40 minutes long), automatically skipping the break window:

```text
Morning block:   08:00–08:40 · 08:40–09:20 · 09:20–10:00
                 10:00–10:40 · 10:40–11:20 · 11:20–12:00

[break:          12:00–13:00  — skipped]

Afternoon block: 13:00–13:40 · 13:40–14:20 · 14:20–15:00
                 15:00–15:40 · 15:40–16:20 · 16:20–17:00
```

Total: **12 slots** generated.

<CodeGroup>
```bash cURL — full workday with lunch break
curl -X POST http://localhost:3000/admin/horarios/masivo \
  -H "Authorization: Bearer <admin_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "id_medico": 5,
    "fecha": "2025-06-10",
    "hora_inicio": "08:00",
    "hora_fin": "17:00",
    "inicio_descanso": "12:00",
    "fin_descanso": "13:00"
  }'
```

```bash cURL — morning only (no break)
curl -X POST http://localhost:3000/admin/horarios/masivo \
  -H "Authorization: Bearer <admin_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "id_medico": 5,
    "fecha": "2025-06-11",
    "hora_inicio": "08:00",
    "hora_fin": "12:00"
  }'
```
</CodeGroup>

**Response `201 Created`**

<ResponseField name="mensaje" type="string">
  Human-readable summary (e.g. `"Se crearon 12 franjas (0 ya existían)."`).
</ResponseField>

<ResponseField name="insertadas" type="integer">
  Number of new slots successfully inserted into the database.
</ResponseField>

<ResponseField name="duplicadas" type="integer">
  Number of slots skipped because they already existed (unique constraint).
</ResponseField>

<ResponseField name="total" type="integer">
  Total slots attempted (`insertadas + duplicadas`).
</ResponseField>

**Error responses**

| Status | Condition |
|--------|-----------|
| `400`  | Missing required fields (`id_medico`, `fecha`, `hora_inicio`, `hora_fin`). |
| `400`  | Time range is too short to fit a single 40-minute slot. |
| `400`  | Break window falls outside the workday boundaries or `inicio_descanso >= fin_descanso`. |
| `404`  | Doctor not found or inactive. |

### Update a Slot

**`PUT /admin/horarios/:id`**

Updates the date and/or times of an existing free slot. The updated slot must be exactly **40 minutes** long and must not overlap with any other slot for the same doctor on the same date.

<ParamField path="id" type="integer" required>
  ID of the slot to update.
</ParamField>

<ParamField body="fecha" type="string" required>
  New date for the slot, format `YYYY-MM-DD`.
</ParamField>

<ParamField body="hora_inicio" type="string" required>
  New start time, format `HH:MM`.
</ParamField>

<ParamField body="hora_fin" type="string" required>
  New end time, format `HH:MM`. Must be exactly 40 minutes after `hora_inicio`.
</ParamField>

<Warning>
  A slot linked to an **active appointment** (`disponible = FALSE`) cannot be edited. Attempting to do so returns `400 Bad Request`.
</Warning>

<CodeGroup>
```bash cURL
curl -X PUT http://localhost:3000/admin/horarios/101 \
  -H "Authorization: Bearer <admin_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "fecha": "2025-06-10",
    "hora_inicio": "10:00",
    "hora_fin": "10:40"
  }'
```
</CodeGroup>

**Response `200 OK`**

<ResponseField name="mensaje" type="string">
  `"Franja actualizada correctamente."`
</ResponseField>

<ResponseField name="franja" type="object">
  The updated slot object with the same shape as the `franja` returned by `POST /admin/horarios`.
</ResponseField>

**Error responses**

| Status | Condition |
|--------|-----------|
| `400`  | Slot has an active appointment (`disponible = FALSE`) and cannot be modified. |
| `400`  | Duration is not exactly 40 minutes. |
| `400`  | `hora_inicio >= hora_fin`. |
| `404`  | Slot not found. |
| `409`  | The new time window overlaps with an existing slot for the same doctor on the same date. |

## Specialty Management

Administrators control the specialty catalog that doctors are associated with and patients browse when booking appointments.

### Create a Specialty

**`POST /admin/especialidades`**

<ParamField body="nombre" type="string" required>
  Unique name of the specialty (e.g. `"Cardiología"`). Duplicate names return `409`.
</ParamField>

<ParamField body="descripcion" type="string">
  Short description shown to patients.
</ParamField>

<ParamField body="precio_base" type="number" required>
  Base consultation price for this specialty.
</ParamField>

<ParamField body="imagen_url" type="string">
  URL or path of the specialty's image. Stored as `imagen_url` in the database.
</ParamField>

<CodeGroup>
```bash cURL
curl -X POST http://localhost:3000/admin/especialidades \
  -H "Authorization: Bearer <admin_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "nombre": "Cardiología",
    "descripcion": "Diagnóstico y tratamiento de enfermedades del corazón.",
    "precio_base": 120.00,
    "imagen_url": "https://cdn.example.com/icons/cardiology.svg"
  }'
```
</CodeGroup>

**Response `201 Created`**

<ResponseField name="mensaje" type="string">Confirmation message.</ResponseField>
<ResponseField name="especialidad" type="object">The newly created specialty record.</ResponseField>

**Error responses**

| Status | Condition |
|--------|-----------|
| `409`  | A specialty with the same `nombre` already exists. |

## Medication Management

Administrators maintain the medication catalog used by doctors when creating prescriptions and clinical records. The admin medication routes use `nombre_comercial`, `principio_activo`, and `presentaciones` field names directly (unlike the public `POST /medicamentos` route which uses short aliases).

### Create a Medication

**`POST /admin/medicamentos`**

<ParamField body="nombre_comercial" type="string" required>
  Commercial name of the medication. Stored directly as `nombre_comercial`.
</ParamField>

<ParamField body="principio_activo" type="string" required>
  Active ingredient(s). Stored directly as `principio_activo`.
</ParamField>

<ParamField body="tipo" type="string">
  Medication class. Accepted values: `OTC` (over-the-counter), `Rx` (prescription-only). Defaults to `OTC`.
</ParamField>

<ParamField body="laboratorio" type="string">
  Name of the pharmaceutical laboratory.
</ParamField>

<ParamField body="categoria" type="string">
  Legacy text category label (optional).
</ParamField>

<ParamField body="descripcion" type="string">
  Free-text description of the medication.
</ParamField>

<ParamField body="indicaciones" type="string">
  Clinical indications.
</ParamField>

<ParamField body="contraindicaciones" type="string">
  Contraindications.
</ParamField>

<ParamField body="presentaciones" type="string">
  Physical form / presentations (e.g. `"Tabletas 500 mg"`). Stored directly as `presentaciones`.
</ParamField>

<ParamField body="registro_invima" type="string">
  INVIMA registration number. Must be unique when provided.
</ParamField>

<ParamField body="imagen_url" type="string">
  URL of the medication's product image.
</ParamField>

<CodeGroup>
```bash cURL
curl -X POST http://localhost:3000/admin/medicamentos \
  -H "Authorization: Bearer <admin_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "nombre_comercial": "Ibuprofeno 400",
    "principio_activo": "Ibuprofeno",
    "tipo": "OTC",
    "descripcion": "Antiinflamatorio no esteroideo de uso general.",
    "presentaciones": "Tabletas 400 mg",
    "laboratorio": "Genfar"
  }'
```
</CodeGroup>

**Response `201 Created`**

<ResponseField name="mensaje" type="string">
  `"Medicamento creado."`
</ResponseField>

<ResponseField name="medicamento" type="object">
  The newly created medication record as stored in the database.
</ResponseField>

**Error responses**

| Status | Condition |
|--------|-----------|
| `400`  | `nombre_comercial` or `principio_activo` missing. |
| `409`  | The `registro_invima` value already exists in the database. |

### Update a Medication

**`PUT /admin/medicamentos/:id`**

Accepts the same body fields as `POST /admin/medicamentos` and replaces the existing record.

<ParamField path="id" type="integer" required>
  ID of the medication to update.
</ParamField>

<ParamField body="nombre_comercial" type="string">
  Updated commercial name.
</ParamField>

<ParamField body="principio_activo" type="string">
  Updated active ingredient.
</ParamField>

<ParamField body="tipo" type="string">
  Updated medication class: `OTC` or `Rx`.
</ParamField>

<ParamField body="laboratorio" type="string">
  Updated laboratory name.
</ParamField>

<ParamField body="descripcion" type="string">
  Updated description.
</ParamField>

<ParamField body="indicaciones" type="string">
  Updated clinical indications.
</ParamField>

<ParamField body="contraindicaciones" type="string">
  Updated contraindications.
</ParamField>

<ParamField body="presentaciones" type="string">
  Updated presentations.
</ParamField>

<ParamField body="registro_invima" type="string">
  Updated INVIMA number.
</ParamField>

<ParamField body="imagen_url" type="string">
  Updated product image URL.
</ParamField>

<ParamField body="activo" type="boolean">
  Active status of the medication. Defaults to `true` if omitted.
</ParamField>

<CodeGroup>
```bash cURL
curl -X PUT http://localhost:3000/admin/medicamentos/12 \
  -H "Authorization: Bearer <admin_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "nombre_comercial": "Ibuprofeno 600",
    "principio_activo": "Ibuprofeno",
    "presentaciones": "Tabletas 600 mg",
    "tipo": "Rx"
  }'
```
</CodeGroup>

**Response `200 OK`**

<ResponseField name="mensaje" type="string">Confirmation message.</ResponseField>
<ResponseField name="medicamento" type="object">The updated medication record.</ResponseField>

**Error responses**

| Status | Condition |
|--------|-----------|
| `404`  | No medication found with the given ID. |

### Toggle Medication Status

**`PATCH /admin/medicamentos/:id/estado`**

Toggles the `activo` boolean of a medication. Deactivated medications remain in the database for historical prescription records but are hidden from active catalog views.

<ParamField path="id" type="integer" required>
  ID of the medication to toggle.
</ParamField>

<CodeGroup>
```bash cURL
curl -X PATCH http://localhost:3000/admin/medicamentos/12/estado \
  -H "Authorization: Bearer <admin_token>"
```
</CodeGroup>

**Response `200 OK`**

<ResponseField name="mensaje" type="string">
  `"Medicamento activado."` or `"Medicamento desactivado."`.
</ResponseField>

**Error responses**

| Status | Condition |
|--------|-----------|
| `404`  | No medication found with the given ID. |
